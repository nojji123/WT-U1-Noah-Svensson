Datakommunikation och webbadresser

a)

Klient avser den personen alternativt det tinget som ansluter till en databas. Servern är enheten som tar emot klientens förfrågningar om att ansluta till databasen.

b)

TCP börjar med att dela upp meddelandet som ska skickas (tex. en hemsida) i olika så kallade informationspaket. Detta gör den för att den ska kunna skicka meddelandet i olika vägar och på så sätt inte belasta enskilda vägar. Sedan numreras de olika paketen och skickas vidare till IP protokollet

IP tar emot det uppdelade TCP Paketet ifrån de olika routrarna samt adresserar det med en så kallad ip adress.

Meddelandet skickas sedan vidare i olika bitar genom olika routrar för att effektivisera leveransarbetet. När paketet sedan anlänt till TCP igen så sorteras det ihop till samma fullständiga paket som det var när det skickades iväg.

c)

Http Är webbsidans protokoll, som används för att föra över Html sidan från en server till en klient.
